Introduction Unreadable code can impair program comprehension, and it can lead to the introduction of bugs. Code mainly contains a natural language text, both in comments and identifiers, and it is a specific form of text  [1] .  But  the methods presented to approximate code readability account for only organizational aspects and visual characteristics of source code, like alignment of characters and line length. Source code could be understandable and readable  [2] . These two features are frequently considered equivalent, but they are unique textual features. Readability relationship to the difficulties of the proposed text, during understandability, is associated with the reader's capability to understand the meaning of the text  [3] . Therefore, code assessed as readable might still not be understandable to a non-reader without the essential knowledge or experience to comprehend it. As the study displays, code comprehension and reading  are  the regular activities created by software developers  [4] . Consequently, code readability,  i.e.,  the ease with which a program can send data to a reader, is a specifically desirable characteristic of the source code. Incomprehensible code could both create more challenges for developers when they try to obtain data from the source code (which applies greater software maintenance expenses in the long run) and can better hide bugs  [5] . Automatically searching for unreadable code in large code is essential for many reasons. Practitioners can utilize this data to estimate where they must focus their efforts in code cleaning and enhancement tasks. Analyst, in contrast, could research how readability (or lack thereof)  affects  further software attributes or tasks  [6] . Identifier names interconnect program perceptions to the reader, and are significant essentials in the procedure of program understanding, and, thus, maintenance of the software. Identifier names are also  constituting  artefacts of the intellectual steps of the programmer, and might be seen as a  representation of the programmer’s comprehension  of the concepts involved in the described solution in the source code  [7] . That low-quality identifier names could be a safety guard to program understandability is enough motivation to make good-quality identifiers. Moreover, there can be broader outcomes of low-quality identifier names. Modern studies on source code readability validated a link between less readable source code and software flaws. Therefore, the study  avoids  the quality of identifier names  [8] . Provided the significance of source code readability of identifier names, we hypothesize that the quality of identifier names  is relative to the software quality  [9] .  The readability metric is consequent from the textual and typographical characteristics of source code, like the usage of whitespace, the total number of brackets and line length, and parentheses  [10] . The readability metric  doesn’t  measure the identifier names quality; it archives only the frequency and length of identifiers, without considering typography or natural language content  [11] . Current studies on source code readability concentrate on the involvement of the modules of source code in readability, and the method in which the semantic content of identifiers contributes to readability and understandability  [12] . The modern computing power, convenience of data, and novel algorithms have led to major achievements in artificial intelligence (AI)  currently . In this context, an additional potential application area for AI/ML is automated code analysis and intelligent software study with multiple motives like vulnerability recognition, code classification, code summarization, and clone identification  [13] . Utilizing ML for software security analysis not only decrease the relies on field specialists for the feature extraction or hand-made pattern, but also supports to ease and automate procedures that are compulsory for the existing security analysis methods. 1.1. Conclusio n In this article, the  IRAF -XADL model has been presented for assessing the quality of identifier names in Python and C++ source code. To perform that, a  syntax-aware identifier preprocessing  module, depending on language-specific  AST  parsing, is  us ed  for  identifier  extraction,  lexical normalization, and semantic cleaning. From the normalized identifiers, this  paper  c alculates  ten  cognitively and  linguistically grounded readability parameters.   In addition , semantic and contextual representations are  obtained   through   CodeBERT  embedding s, which  are processed by an  SA- BiLSTM   network for  learn ing  sequential and contextual dependencies.   Moreover , the model is optimized  with  the  AdamW  optimizer,  allowing   better  convergence and  enhanced  overall performance.   Finally , the SHAP-based explainability  method  is in corporat ed  to  interpret the contribution of identifier tokens and  attributes  toward readability predictions.   The experimental result analysis of the  IRAF-XADL  model   is  conducted , and the comparative results  highlighted  the  supremacy   of the proposed  system   with an accuracy of  98.13 % and  98.42 % under Python and CPP data   compared to   other   techniques  concerning   different  me asure s .
